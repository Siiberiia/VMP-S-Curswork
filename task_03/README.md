# task_03 — Завод: сборка + переработка + качество (дискретная симуляция)

## Что моделируется

- Есть **AM** сборочных автоматов и **RM** автоматов переработки.
- Рецепт изделия: `n` типов компонентов с количествами `q1..qn`.
- За один дискретный момент времени `t` на склад поступают компоненты **1-го уровня** со скоростями `s1..sn`.
- Уровни качества компонентов/изделий: `1..5` (обычный → легендарный).

### Время цикла и модули качества

Базовые длительности цикла: `t_assembly`, `t_recycle` (целые дискретные единицы времени).

Если на типе автомата установлено `m` модулей (`m in [0..4]`), то время цикла:

`T = round(t_base * (1 + 0.1*m))`

(каждый модуль добавляет **10%** к базовому времени, как «аддитивное замедление» относительно базы).

### Вероятности повышения качества

Для прыжка на `+k` (где `k>=1`) одним модулем задано:

`P_k = 52 * 10^(-2-k)`

Если модулей `m`, то массы для `k>=1` **складываются**: `P_k(m) = m * 52 * 10^(-2-k)`.

Оставшаяся масса — «остаться на том же уровне» (`+0`).

Если сумма `> 1`, нормализуем массы для `k>=1` и остаток отдаём «остаться».

### Сборка

Если автомат свободен и на складе хватает компонентов по рецепту, он **забирает** компоненты (жадно: сначала более низкие уровни, чтобы не «сжигать» редкие раньше времени).

Базовый уровень изделия:

`L = min(уровней реально взятых единиц компонентов)`

Затем выбирается приращение `k` по закону модулей сборки, итоговый уровень изделия:

`L_out = clamp(L + k, 1..5)`

### Переработка

Если автомат свободен и есть готовые изделия, берётся изделие **минимального доступного уровня** (сначала «обычные»).

За цикл переработки по каждому типу компонента `i` возвращается примерно **25%** рецепта в дискретных единицах:

`add_i = ceil(q_i / 4) = (q_i + 3) / 4` (целочисленно)

Уровень возвращаемых компонентов:

`C_out = clamp(L + k, 1..5)`, где `k` — тот же тип случайного приращения, но по модулям **переработки**.

## Условие остановки

Симуляция заканчивается, как только произведено **25** изделий **5-го уровня** (легендарных).

## Запуск (командная строка)

```text
task_03 AM RM t_assembly t_recycle n q1..qn s1..sn [assembly_modules recycle_modules]
```

Пример:

```bash
./task_03 2 1 3 4 2 2 3 5 5 1 1
```

## Выходные данные

В текущей директории создаются файлы:

- `times_L3.txt` — моменты времени, когда сборка произвела изделие уровня 3
- `times_L4.txt` — то же для уровня 4
- `times_L5.txt` — то же для уровня 5

В консоль печатается краткая сводка.
