# task_13 — Случайный граф: ожидания и дисперсии (Монте-Карло)

## Модель

Генерируется **простой неориентированный** случайный граф **G(n, p)**:

- для каждой пары вершин `(u, v)` ребро присутствует с вероятностью `p` (независимо);
- каждому ребру независимо присваивается вес из `{1, 2, …, 10}` (равномерно).

Дальше по каждому реализованному графу считаются величины из задания, по серии графов оцениваются **выборочное среднее** и **выборочная дисперсия** (алгоритм Уолфорда).

## Что считает программа

| Обозначение | Смысл |
|-------------|--------|
| **A** | Сумма весов **MST** (алгоритм Краскала), только если граф **связный** (несвязные реализации в статистику A не входят). |
| **B** | Число рёбер в **самом длинном** (по числу рёбер) **простом цикле**; если циклов нет — `0`. |
| **D** | Среди простых циклов выбирается **максимальный суммарный вес**; если таких несколько — берётся цикл с **наибольшим** числом рёбер; если циклов нет — `0`. |
| **E** | Число **изолированных** вершин. |
| **F** | Число **неизоморфных** деревьев на `n` вершинах (последовательность **A000055**); для фиксированного `n` это **константа**: `Var[F] = 0` (в коде таблица для `1 ≤ n ≤ 25`). |
| **G** | Число **компонент связности**. |
| **H** | Число компонент связности, которые являются **кликами** (полными подграфами) в исходном графе; изолированная вершина считается кликой `K₁`. |

### Про циклы (важно для времени работы)

Перебор всех простых циклов — задача с **экспоненциальной** сложностью в худшем случае. Для **плотных** графов и больших `n` программа может работать долго.

Рекомендации для курсовой и демонстрации:

- `n ≤ 10`, `p` около `0.25 … 0.45`, `trials` от нескольких тысяч.

Формат:

```text

```

- `n` — число вершин;
- `trials` — число случайных графов;
- `p` — вероятность ребра (по умолчанию `0.35`).

## Что написать в отчёте

- описание модели `G(n, p)` и распределения весов;
- алгоритмы: MST (Краскал), поиск компонент (DFS), проверка клики в компоненте;
- для **B** и **D**: канонизация циклов (каждый простой цикл учитывается при минимальной вершине `s`, дальше DFS только в вершины `> s`, замыкание в `s`);
- для **F**: ссылка на OEIS A000055 и пояснение, что это не случайная величина при фиксированном `n`;
- таблица результатов для 2–3 наборов `(n, p, trials)` и краткий вывод, как `p` и `n` влияют на средние.
