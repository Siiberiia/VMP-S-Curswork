# task_07 - Пересечение множеств и связь с шифром Вернама

## Что решается

Пусть есть конечное множество мощности `N`, и два случайных подмножества:
- `X`, `Y`,
- `|X| = |Y| = k`.

Определим:

- `R(k,N) = P(X ∩ Y != Ø)`.

Нужно:
1. Проверить нижнюю оценку  
   `R(k,N) >= 1 - exp(-k^2 / N)`.
2. Найти `k`, при котором `R(k,N) > 1/2`.
3. Применить к 16-битному шифру Вернама.

## Идея и формулы

Точная вероятность отсутствия пересечения:

- `P(X ∩ Y = Ø) = C(N-k, k) / C(N, k)`.

Значит:

- `R(k,N) = 1 - C(N-k, k)/C(N, k)`.

Оценка:

- `R(k,N) >= 1 - exp(-k^2/N)`.

Из оценки для условия `R(k,N) > 1/2`:

- достаточно `1 - exp(-k^2/N) > 1/2`,
- `k > sqrt(N ln 2)`.

Для Вернама (`m` 16-бит) мощность пространства:

- `N = 2^16 = 65536`.

## Что делает программа

`src/main.cpp`:
- считает точное `R(k,N)`;
- считает нижнюю границу `1-exp(-k^2/N)`;
- оценивает `R(k,N)` методом Монте-Карло;
- находит минимальное `k`, где точное `R(k,N) > 1/2`;
- находит минимальное `k`, где граница уже > `1/2`;
- делает демонстрацию для Вернама:
  - `E_t(m) = m XOR t` (фиксированный 16-битный ключ),
  - коллизии шифртекстов между двумя наборами соответствуют пересечениям по открытым текстам.

## Для отчета

В отчет удобно вставить:
- формулу точного `R(k,N)`;
- оценку `R(k,N) >= 1 - exp(-k^2/N)`;
- вывод о пороге `k ~ sqrt(N ln2)`;
- численный результат для `N=65536`;
- пояснение по Вернаму:
  - при повторном использовании одного ключа отображение `m -> E_t(m)` биективно,
  - поэтому совпадения шифртекстов между наборами указывают на совпадения открытых текстов,
  - это и дает практическую идею атаки по совпадениям.
