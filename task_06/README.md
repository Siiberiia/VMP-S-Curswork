# task_06 — Движение точки по идеальному M-арному дереву (Qt)

## Постановка (что моделируем)

Идеальное **M-арное** дерево глубины `depth`:
- корень на уровне `0`;
- каждый внутренний узел имеет ровно `M` потомков;
- все листья находятся на одном уровне `depth`.

Материальная точка стартует из корня и движется по узлам вниз.

Для каждого внутреннего узла (в приложении — по уровням дерева) задано:
- **вероятность поглощения** `p_stop(level)`: попав в узел на уровне `level`, точка может **навсегда остаться** в этом узле;
- если поглощения не произошло, выбирается **следующий потомок** по заданному закону (равномерно или с перекосом).

Требуется:
- экспериментально оценить вероятность попадания в заданный лист;
- оценить вероятность того, что точка пройдёт `L` вершин (число посещённых вершин, включая корень);
- визуализировать путь точки в дереве;
- продемонстрировать работу для разных `M`, законов движения и значений `p_stop`.

## Что сделано

Оконное приложение (Qt6 Widgets) в `src/main.cpp`:
- ввод параметров `M`, `depth`, `leafId`, `p_stop` по уровням, закон движения, `trials`;
- кнопка **«Один эксперимент»** — рисует путь синей линией;
- кнопка **«Серия»** — оценивает вероятности методом Монте‑Карло и печатает результаты;
- дерево рисуется на `QPainter` в области со скроллом.

### Как задаётся целевой лист

Лист задаётся как `leafId` в диапазоне `0 .. M^depth - 1`.
Также приложение показывает его представление в системе счисления base‑`M`
в виде последовательности цифр длины `depth` (это и есть путь от корня к листу).

### Как задаются `p_stop` по уровням

Поле `p(остаться) по уровням` — список чисел в `[0,1]` через пробел/запятую:

Пример для `depth=5`:

`0.10 0.05 0.02 0.01 0.00`

Если список короче `depth`, для остальных уровней берётся значение `p(остаться) по умолчанию`.

### Законы движения

- **Равномерно по потомкам**: каждый из `M` потомков выбирается с вероятностью `1/M`.
- **С перекосом в 0-го потомка**: с вероятностью `alpha` выбирается потомок `0`,
  иначе равномерно выбирается один из `1..M-1`.

## Выводимые эмпирические величины

При серии из `trials` экспериментов:
- `P(дойти до любого листа)` — доля экспериментов, в которых точка дошла до уровня `depth`;
- `P(остановиться во внутреннем узле)` — доля поглощений до листа;
- `P(попасть в заданный лист)` — доля экспериментов, где достигнут лист с данным `leafId`;
- распределение `P(L = l)` для `l = 1..depth+1`, где `L` — число посещённых вершин (включая корень).

## Сборка и запуск на Windows (Qt 6.11 MinGW)

PowerShell (пути при необходимости поправьте под свою установку Qt):

```powershell
$env:PATH = "C:\Qt\6.11.0\mingw_64\bin;C:\Qt\Tools\mingw1310_64\bin;C:\Qt\Tools\CMake_64\bin;C:\Qt\Tools\Ninja;" + $env:PATH

cmake -S "C:\CursWorkTV\task_06" -B "C:\CursWorkTV\task_06\build" -G Ninja `
  -DCMAKE_PREFIX_PATH="C:/Qt/6.11.0/mingw_64" `
  -DCMAKE_CXX_COMPILER="C:/Qt/Tools/mingw1310_64/bin/g++.exe"

cmake --build "C:\CursWorkTV\task_06\build"

& "C:\CursWorkTV\task_06\build\task_06.exe"
```

Перед повторной настройкой CMake удалите `build`, если папка создавалась из WSL (`/mnt/c/...`) — пути в кэше должны совпадать с Windows (`C:/...`).
